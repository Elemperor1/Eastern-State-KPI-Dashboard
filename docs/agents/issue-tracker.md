# Issue tracker: GitHub

Issues for this repository live in GitHub Issues at
`Elemperor1/Eastern-State-KPI-Dashboard`. Use the `gh` CLI for operations.

## Conventions

- Create one issue per independently deliverable ticket.
- Read issue bodies, comments, and labels before acting.
- Apply `ready-for-agent` to fully specified tickets.
- Use GitHub's native issue dependencies for blocking relationships.
- If native dependencies are unavailable, add `Blocked by: #<issue>` to the body.
- A ticket is ready only when every blocking issue is closed.
- Do not use pull requests as a triage request surface.

## Publishing

When an engineering skill says to publish a ticket, create a GitHub issue in
this repository and apply the configured triage label.
