# Domain documentation

This is a single-context repository.

Before exploring or changing the codebase:

1. Read the root `CONTEXT.md`.
2. Read the ADRs relevant to the affected area under `docs/adr/`.
3. Use glossary terms in issue titles, tests, and implementation work.
4. Avoid synonyms explicitly rejected by `CONTEXT.md`.
5. Surface any proposed contradiction with an accepted ADR.
6. Keep architecture changes preservation-first and strengthen
   characterization coverage before moving business logic.
